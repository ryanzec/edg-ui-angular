By default, updating a signal that tracks a non-scalar value will trigger an effect to be called even if the value is effectively the same (like setting an array to a new array with the same values).

{{ signalTriggeredCount() }}
You can provide a custom equal function when creating the signal to avoid this issue.

{{ signalTriggeredCustomEqualityCount() }}
